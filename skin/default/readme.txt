◆◆skin仕様◆◆
・最初に
skin.jsonはunicodeのjson形式です。コメント//や/* */は、使用できませんのでご注意ください。
skin.jsonを変更した場合は、アプリ再起動が必要です。

◆◆タブ◆◆
タブskinに指定できるパラメータは次の通りです。
"tab":{
	"img":			"ui/tab.png",				〜 非aeroでのタブイメージファイル(32bitpngファイル)
	"aeroimg":		"ui/tabaero.png",			〜 aeroでのタブイメージファイル(32bitpngファイル)
	"color":		[0,32832,16711680,0,0,0],   〜 非aeroでのタブテキストカラー(RGB 10進値)
												   [通常状態カラー,loadingカラー,未読カラー,activeカラー,tabgroup activeカラー,hotカラー]
	"aerocolor":	[0,32832,16711680,0,0,0],	〜 aeroでのタブテキストカラー、カラー順序は非aeroと同じ
	"maxfontheight":16,							〜 タブイメージで表示可能な最大fontの高さ(point指定)
	"height":		24,							〜 タブイメージの高さ(pixel指定)
	"graw":			15,							〜 aeroでのテキストの周りのgraw値
	"activebold":	true						〜 タブがactiveのときboldフォントで表示するときtrue
},

◆◆ツールバーボタン◆◆
ツールバーのイメージには、
  通常タイプ
  カスタムタイプ
があります。

通常タイプとカスタムタイプの違いは、通常タイプは標準のwindowsの機能により描画されます(hotや押されたときなど)。
カスタムタイプは、用意した画像がそのまま描画されます(押された時なら、押された時の画像がそのまま描画)。

通常タイプは、各サイズ
  小	16×16
  中	24×24
  大	32×32
  特大	48×48
のpng画像をbtnフォルダ下の「コマンド名+ボタンサイズ.png」に置きます。
サイズの画像がない場合は、アプリが存在する画像をリサイズして使用します。

カスタムタイプは、refreshボタンを例に説明すると、
"btn":{
	・・・ 他の設定
	"refresh":			{"img":"btn/refresh.png","custom":true},
	・・・ 他の設定
}
と、"custom":trueとイメージパスをskin.jsonに指定する必要があります。
"custom":trueとなっているものがカスタムアイコン描画されます。
customがついていないものは、通常タイプで描画されます。

カスタムタイプの画像ファイルは、
  ・小サイズ  ：btn/refrssh16.png (実際に用意する画像サイズ 縦22×横88)
  ・中サイズ  ：btn/refresh24.png (実際に用意する画像サイズ 縦30×横120)
  ・大サイズ  ：btn/refresh32.png (実際に用意する画像サイズ 縦38×横152)
  ・特大サイズ：btn/refresh48.png (実際に用意する画像サイズ 縦54×横216)
を用意します。全てを用意する必要は無く、アプリは、表示するサイズの画像が無い場合は、
存在するサイズの画像をリサイズして使用します。

小サイズだと、カスタムタイプ画像の内容は、22×22を1単位として
  ・1 通常時
  ・2 マウスカーソルオーバー時(ボタンhot表示)
  ・3 ボタンpush状態
  ・4 ボタン無効状態
です。
